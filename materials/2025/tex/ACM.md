# ハッカーズ大辞典

ハッカーたちが使う用語を集めた辞典に、エリック・S・レイモンド編、福崎俊博訳、
『ハッカーズ大辞典 改訂新版(英語版の第三版の訳)』、 ASCII、 2002
年がある。

この辞典、もともとは Jargon File というインターネットで作っていた
用語集の ver. 4.0.0 を本にして出版したもので、 現在のバージョン(ver.
4.3.3 )はエリック・S・レイモンドの Jargon File Resources ページ
`http://www.tuxedo.org/esr/jargon/` で全部読むことができる。

この辞典には、なんだかほとんど意味の不明なものから、非常に興味深いお話まで、
実に幅広い話題が載っている。 たとえば C
の開発者の一人、ブライアン・カーニハンが Pascal
のことを「ボンテージ調教言語」だとこきおろした話とか、 「AI
公案」のところに書いてある LISP マシンがうまく動かなくて
電源を切ったり入れたりしている人のところへ、その開発者の一人の
トム・ナイトがやってきて「原因もわからないで電源を入れ直しても
絶対直らんよ」と言いつつ、当の本人は単に電源を切って入れ直して、
しかもそれでマシンが直ってしまったというお話など、おもしろ
おかしく紹介されている。

今回はそのうちから「バックドア」の項を紹介しよう。

# バックドア

マシンのセキュリティの問題は、インターネット常時接続が当たり前になって、
自前でサーバを建てたり、クラッキング・ツールが出回ったり、
ウィルスが流行していることなどもあり、最近、話題になっているものの一つである。
ちょっと前だったら信じられないことだが、セキュリティ関係の雑誌も何冊も創刊されている。

さて、 UNIX はインターネットのサーバとして活用されていて、 基本的に
Windows などよりはセキュリティがずっと高い OS ではあるが、
それはあくまでもちゃんと手入れが行き届いていればのお話である。
全体のセキュリティは、結局のところ、一番低いところと同じになって
しまうのだ。
既に知られているセキュリティ・ホールを修正しないで放っておけば、
どこからともなくチェックされて、バックドア(裏の侵入口)をしかけられて、
知らないうちに他のマシンへの攻撃用マシンに改造されてしまうことうけあい
である。

# 「信頼を信頼することについて」

では、『ハッカーズ大辞典』の「バックドア」の項目を見てみよう。

`http://www.tuxedo.org/esr/jargon/html/entry/back-door.html`

Web ページの方は英語で書かれてので、簡単にその内容を紹介する。
ここで取り上げられているのは、 UNIX の開発者の一人、ケン・トンプソン
(スペース・トラベルで遊びたかった人)のお話である。

ケン・トンプソンは UNIX の開発によって、コンピュータの世界のノーベル賞に
あたる ACM チューリング賞を 1983 年に受賞した。
その記念講演がすごかった。 タイトルは「Reflections on Trusting
Trust(信頼を信頼することについて)」。
`http://www.acm.org/classics/sep95/` にその全文(英語)がある。

「このような賞をくださった ACM に感謝します。」ではじまるこの講演は、
UNIX に関して話すには、最近は自分よりももっとふさわしい人がいっぱいいる
だろうし、
「わたしはプログラマーです。税金の申告用紙の職業欄にもそう書きます。」
ということで、「わたしが書いた、生涯において最高に
キュートなプログラムについて お話しましょう」ということになった。

この話は 3 つのステージに沿って進められた。 その
1、自分自身を作り出すプログラムが書けること
(要するに実行すると「`hoge.c`」を作るようなプログラム
「`hoge.c`」を書くことができること)。 その
2、ニワトリが先かタマゴが先かみたいな話だが、 C コンパイラも C
言語で書かれていること。 その 3、つまり 1 と 2 を総合すると C
コンパイラに仕掛けすることができて、 C
コンパイラ自身が「トロイの木馬」タイプのウィルスのようなものになり得ること
・・・。 そして、
「モラルは明白です。自分で最初から全部作ったコード以外は信じることなんてできないのです
(特に私のような人間を雇っている会社のコードなんてね)。」と続け、
遊びでクラッキングをしかけるようなガキに関してちゃんと考えた方がいいだろうという話で
講演を終わっている。

これが、 UNIX
に関する記念講演で話されたこと、自分で書いたサイコーにキュートな
プログラムだと言っていること。
さて、これがどんな意味を持つかわかるだろうか？

つまり、初期の UNIX のあるバージョンのログインのプログラムには、 某 K.
T. くんのログインを許可するようなバックドアが仕掛けられていた。
これは、ソースが公開されている[^1] UNIX
では、バックドアのない版をコンパイルしなおしてインストールすれば、
普通はバックドアは取り除ける。 でも、実はそのコンパイルをするための C
コンパイラ自体にもしかけがあって、
バックドアがついていないログインのプログラムに、
なんと勝手にバックドアを埋め込んだり、 C
コンパイラ自体も同様の仕掛けがされていたというのである・・・。

ケン・トンプソンは、自分の若き日の経験をふまえ、
クラッカーのガキをいかにもスゴイ連中のように
取り上げるマスコミに注意するように言っている。
そもそも、クラッカーのガキどもは、自分が何をしているのか、
そしてそれを行った結果、どれだけの責任
(刑務所に長い期間閉じ込められるとか、ずっと犯罪者の烙印を押されるとか)
をしょいこむことになるか、わかっちゃいないのだと。
そして、インターネットが普及して、他人のコンピュータがつながりあっている
今日、コンピュータの濫用は、飲酒運転よりも起こりうる重大な問題である
ことを深く認識するべきであろうと。

[^1]: 普通に考えれば当たり前だが、
    公開されている版のソースからはしっかりバックドアの部分は取り除かれている。
