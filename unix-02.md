# UNIXシステムを使ってみよう：最初のコマンド

前の章では、UNIXシステムの概要について学んだ。この章では、いよいよ実際にUNIXシステムに触れて、基本的なコマンドを使ってみる。コマンド操作は、初めは少し難しく感じるかもしれないが、慣れてくると非常に効率的にコンピュータを操作できるようになる。

## ターミナルを開いてみよう

UNIXシステムでコマンドを入力するには、「ターミナル」というアプリケーションを使う。ターミナルは、キーボードから命令（コマンド）を打ち込み操作するためものである。

で検索を起動し、「ubuntu」で検索して「Ubuntu」を起動する。

*   **macOS**: Finder <img src="images/AppIconTopic_Finder.png" width="20px" alt="Finder" /> で アプリケーション ＞ ユーティリティ ＞ ターミナルと辿ってダブルクリックして起動する。<img src="images/AppIconTopic_Launchpad.png" width="20px" alt="Launchpad" />で検索して起動してもいい。

### プロンプト

ターミナルを起動すると、次のような画面が表示される。

**ターミナルでWSL上のUbuntu:**
```sh
ユーザー名@コンピュータ名:~$
```
```sh
(仮想環境名) ユーザー名@コンピュータ名:~$
```

**macOSのターミナル:**
```zsh
ユーザー名@コンピュータ名 ~ %
```
```zsh
(仮想環境名) ユーザー名@コンピュータ名 ~ %
```

これらの表示を「**プロンプト** (prompt)」と呼ぶ。プロンプトは、「入力促進記号」と呼ばれ、「コマンドを入力してください」というシステムからのメッセージである。このプロンプトに続けて、様々なコマンドを入力していく。

なお、この資料では、macOSのプロンプトを `%`、Ubuntuのプロンプトを `$` と表記する。プロンプトの表示形式はカスタマイズ可能だが、ここでは基本的な表示で説明を進める。

## 最初のコマンド：システム情報を表示する `uname`

それでは、最初のコマンドとして `uname` を使ってみよう。`uname` は "**u**nix **name**" の略で、現在使用しているUNIXシステムに関する情報を表示するコマンドである。

プロンプトに続けて、`uname -a` と入力し、Enterキーを押す。`-a` は「すべての情報を表示する」という意味のオプションである。

```zash
% uname -a
```

**WSL上のUbuntuの場合の実行例:**
```bash
$ uname -a
Linux HOSTNAME 5.15.167.4-microsoft-standard-WSL2 #1 SMP Thu Nov 5 00:21:55 UTC 2024 x86_64 x86_64 x86_64 GNU/Linux
$
```

**macOSの場合の実行例:**
```zsh
% uname -a
Darwin MacBook-Air.local 24.4.0 Darwin Kernel Version 24.4.0: Fri Apr 11 18:32:50 PDT 2025; root:xnu-11417.101.15~117/RELEASE_ARM64_T6041 arm64
%
```

表示される内容は複雑に見えるかもしれないが、ここにはOSの種類、バージョン、CPUの種類などが含まれている。
例えば、macOSの例では、`Darwin` がOSのカーネル（核）の名前、`24.4.0` がそのバージョン、`arm64` がCPUアーキテクチャ（Apple Siliconなど）を示している。
WSL上のUbuntuの例では、`Linux` がカーネル名、`5.15.167.4-microsoft-standard-WSL2` がそのバージョン、`x86_64` がCPUアーキテクチャ（一般的なIntel/AMD系CPU）を示している。
このように、コマンドに「オプション」と呼ばれる指示を追加することで、コマンドの動作を細かく制御できる。

### コマンド入力の基本とトラブルシューティング

コマンド入力に慣れないうちは、いくつか戸惑うことがあるかもしれない。よくあるケースと対処法を覚えておこう。

*   **大文字・小文字の区別**: UNIXコマンドやファイル名では、大文字と小文字は厳密に区別される。`uname` を `Uname` や `UNAME` と入力しても正しく動作しない。
*   **入力ミス**: タイプミスは誰にでもある。Enterキーを押す前であれば、左右キーで前後したり、`Delete`キーや`Backspace`キーで消すことができる。入力を中断するには `Ctrl + c` (コントロールキーを押しながらcキー)である。
*   **`command not found` エラー**: コマンド名が間違っていたり、システムにそのコマンドがインストールされていなかったりすると、「command not found」（コマンドが見つかりません）というエラーメッセージが表示される。まずはコマンドのスペルを確認しよう。
    ```zsh
    % unmae -a
    zsh: command not found: unmae
    ```
*   **コマンドが反応しない・暴走した？**: 意図しないコマンドを実行したり、プログラムが応答しなくなったりときには、`Ctrl + c` を押してみる。これは実行中のコマンドを強制的に中断する操作で、多くの場合これでプロンプトが表示される。

### `uname` コマンドのオプション

`uname` コマンドには、`-a` 以外にも様々なオプションがある。オプションは通常、ハイフン (`-`) に続けて1文字以上の英字で指定する。いくつか試してみよう。

*   **カーネル名を表示 (`-s`)**: OSの中核部分であるカーネルの名前を表示する。
    ```sh
    % uname -s
    Darwin  # macOSの場合
    $ uname -s
    Linux   # Ubuntuの場合
    ```
*   **カーネルのリリースバージョンを表示 (`-r`)**: カーネルのバージョン番号を表示する。
    ```zsh
    % uname -r
    24.4.0  # macOSの場合
    $ uname -r
    5.15.167.4-microsoft-standard-WSL2 # Ubuntuの場合
    ```
*   **マシンのハードウェアタイプを表示 (`-m`)**: コンピュータのCPUの種類などを表示する。
    ```zsh
    % uname -m
    arm64  # Apple Silicon搭載Macの場合
    $ uname -m
    x86_64 # 一般的なIntel/AMD系CPUのPCの場合
    ```

オプションを指定しない場合は、`-s` と同じくカーネル名が表示される。

```sh
uname [オプション]
```

このように、コマンドの後ろに付けるオプションやファイル名などを「**引数**（ひきすう、argument）」と呼ぶ。これはプログラミングやマニュアルなどでもよく使われる用語なので、覚えておこう。

> ### ☆練習☆
>
> 1.  `uname -a` を実行し、表示された情報の中から、使っているシステムのカーネル名、カーネルバージョン、ハードウェアタイプがどれにあたるか確認してみよう。
> 2.  WSL上のUbuntuとmacOS（またはその他のUNIX系のシステム）で、`uname` コマンドの各オプション (`-s`, `-r`, `-m`) の実行結果を比較し、どのような違いがあるか観察してみよう。

## その他の簡単なコマンド

`uname` 以外にも、UNIXシステムにはたくさんの便利なコマンドがある。いくつか基本的なものを紹介する。

### 日時を表示する: `date` コマンド

`date` コマンドは、現在の日付と時刻を表示する。`date` コマンドに以下のようなオプションを付けて実行してみよう。

```zsh
% date "+%Y-%m-%d %H:%M:%S"
2025-05-07 14:30:15
```

`date` コマンドのオプションでは、表示形式を細かく指定できる。オプションは `+` に続けて書式指定文字を記述する。例えば、`"+%Y-%m-%d %H:%M:%S"` というオプションを付けると、`年-月-日 時:分:秒` の形式で表示される。これはプログラムで日時を扱う際などによく使われる形式である。

日本語の曜日やタイムゾーンを含めて表示することもできる。

```zsh
% date "+%Y年%m月%d日 (%A) %H時%M分%S秒 %Z"
2025年05月07日 (水曜日) 14時30分15秒 JST
```

主な書式指定文字：
*   `%Y`: 西暦年 (4桁)
*   `%m`: 月 (01-12)
*   `%d`: 日 (01-31)
*   `%A`: 曜日名 (例: Sunday, Monday, ..., 日曜日, 月曜日, ...)
*   `%a`: 短縮された曜日名 (例: Sun, Mon, ..., 日, 月, ...)
*   `%H`: 時 (00-23)
*   `%M`: 分 (00-59)
*   `%S`: 秒 (00-59)
*   `%Z`: タイムゾーン名 (例: JST, UTC)

### ログインしているユーザーを表示する: `who` コマンド

`who` コマンドは、現在そのコンピュータにログインしているユーザーの一覧を表示する。UNIXシステムは複数のユーザーが同時に利用できるため、このようなコマンドがある。

```zsh
% who
username console  May  7 09:15
otheruser ttys001 May  7 11:30 (192.168.1.10)
```

表示内容は、ユーザー名、利用している端末（consoleやttys001など）、ログイン日時、接続元IPアドレス（リモート接続の場合）などである。

また、`whoami` (who am i) というコマンドは、現在操作しているユーザー自身の名前を表示する。

```zsh
% whoami
ユーザー名
```

### Webページの内容を取得する: `curl` コマンド

`curl` (カール) は、指定したURLからデータを取得するコマンドである。Webページの情報を取得したり、ファイルをダウンロードしたりするのに使える。例えば、天気予報サイトの情報を見てみよう。

```zsh
% curl -s "https://wttr.in/Tokyo?format=%l:+%c+%t"
Tokyo: ☀️ +25°C
```

`curl` の `-s` オプションは、進捗表示などを省略して結果のみを表示する指定である。`curl` は非常に多機能で、Web APIを利用する際などにも強力なツールとなる。`Tokyo` の部分を `Osaka` や `London` など、他の都市名に変えて試してみることもできる。

### 文字列を表示する: `echo` コマンド

`echo` コマンドは、指定した文字列を画面に表示するシンプルなコマンドである。メッセージを表示したり、他のコマンドと組み合わせて使ったりする。

```zsh
% echo "こんにちは、UNIXの世界へようこそ！"
こんにちは、UNIXの世界へようこそ！
```

## コマンド操作で困ったときの対処法まとめ

コマンド操作中に困ったときの対処法を改めて整理しておく。

1.  **入力ミス・修正**: Enterキーを押す前なら左右キーで移動し、 `Delete` や `Backspace` で修正できる。入力を取り止めたい場合は `Ctrl + c`。
2.  **`Command not found`**: コマンドのスペルミスがないか確認。ミスがなくてもダメな場合、コマンドがインストールされていない可能性もある。
3.  **コマンドが暴走・応答なし**: `Ctrl + c` で強制中断。
4.  **コマンド実行後、プロンプトが出ない**: GUIアプリケーションなどを起動した際に、ターミナルがそのアプリケーションの終了を待っている状態になることがある。その場合は、アプリケーションを終了させるか、`Ctrl + z` で一時停止し `bg` コマンドでバックグラウンド実行に切り替えるなどの対処がある（詳細は後の章で）。
5.  **対話型プログラムの終了方法**: `Ctrl + d` (ファイルの終わりを意味する) で終了できるプログラムが多い。プログラムによっては `quit` や `exit` と入力して終了するものもある。
    例えば、`bc` という電卓コマンドを起動してみよう。
    ```zsh
    % bc
    bc 1.07.1
    Copyright 1991-1994, 1997, 1998, 2000, 2004, 2006, 2008, 2012-2017 Free Software Foundation, Inc.
    This is free software with ABSOLUTELY NO WARRANTY.
    For details type `warranty'.
    1+2*3
    7
    quit  # または Ctrl + d
    %
    ```
6.  **画面の表示が乱れた (文字化けなど)**: ターミナルの表示がおかしくなった場合は、`Ctrl + l` (小文字のL) で画面をクリアできる。

## この章で学んだコマンド

| コマンド | 読み方   | 説明                                       | 主な使い方                         | 例                                       |
| :------- | :------- | :----------------------------------------- | :--------------------------------- | :--------------------------------------- |
| `uname`  | ユーネーム | システム情報を表示する                     | `uname [オプション]`               | `uname -a` (すべての情報を表示)          |
| `date`   | デート   | 現在の日付と時刻を表示する                 | `date ["+書式"]`                   | `date "+%Y-%m-%d"` (年-月-日形式で表示) |
| `who`    | フー     | ログインしているユーザーの一覧を表示する     | `who`                              | `who`                                    |
| `whoami` | フーアムアイ | 現在操作しているユーザー名を表示する         | `whoami`                           | `whoami`                                 |
| `curl`   | カール   | URLを指定してデータを取得する              | `curl [オプション] "URL"`          | `curl -s "https://wttr.in/Tokyo"`      |
| `echo`   | エコー   | 指定した文字列を画面に表示する             | `echo "文字列"`                    | `echo "Hello"`                         |
| `bc`     | ビーシー | コマンドライン電卓（対話型）を起動する       | `bc` (終了は `quit` または `Ctrl+d`) | `bc`                                     |

この章では、UNIXシステムを操作するための第一歩として、ターミナルの使い方と基本的なコマンドをいくつか紹介した。最初は覚えることが多く感じるかもしれないが、実際に手を動かしながら少しずつ慣れていこう。
