# 長い表示も快適に：ページャー `less` の使い方

前の章では、ファイルの作成やコピー、削除といった基本的なファイル操作を学んだ。しかし、ファイルの中身が長かったり、コマンドの実行結果がたくさん表示されたりすると、画面がスクロールしてしまい、先頭部分が見えなくなることがある。また、「このコマンドの詳しい使い方が知りたい」と思うこともあるだろう。

この章では、そのような場合に役立つ「ページャー」という種類のコマンドを紹介する。

## 1画面ずつ表示する: ページャー `less` (レス)

### ページャーとは何か？

例えば、`cat` コマンドで長いファイルの中身を表示しようとすると、あっという間に画面が流れてしまい、最初の方を読むことができない。そのようなとき、「ページャー」を利用すると、ファイルの内容を1画面の大きさに区切って表示し、まるで本を1ページずつめくるように閲覧できる。

UNIXシステムでよく使われるページャーには `more` (モア) と `less` (レス) がある。ここでは、より高機能な `less` を紹介する。`less` は macOS や WSL/Linux (Ubuntuなど) に標準でインストールされているため、すぐに利用できる。

### `less` でファイルを見てみよう

まず、`less` で表示するための少し長めのファイルを用意する。ここでは例として、いくつかのコマンドの実行結果を一つのファイルにまとめてみる。

```bash
# 現在の日時を date.txt に保存
% date "+%Y-%m-%d %H:%M:%S" > date.txt

# システムの情報を system_info.txt に保存
% uname -a > system_info.txt

# ディスクの空き容量を disk_space.txt に保存
% df -h > disk_space.txt

# 上の3つのファイルの内容をまとめて long_text.txt というファイルに保存
% cat date.txt system_info.txt disk_space.txt > long_text.txt
```

これで、`long_text.txt` という少し長めのファイルが作成された。このファイルを `less` で表示してみよう。

`less` を使ってファイルを表示するには、次のように入力する。

```bash
less 「ファイル名」
```

では、`less long_text.txt` と入力して実行する。

```bash
% less long_text.txt
```

すると、`long_text.txt` の内容が画面いっぱいに表示され、一番下にはファイル名（この場合は `long_text.txt`）やコロン `:` が表示されているはずである。これが `less` の表示画面である。

### `less` の基本的な操作

`less` を使っている間の主な操作はキーボードで行う。

| 動作                                  | キー                     | 備考                                   |
| :------------------------------------ | :----------------------- | :------------------------------------- |
| 次のページへ進む                      | `スペース` キー または `f` |                                        |
| 前のページへ戻る                      | `b` キー                 |                                        |
| 1行下へスクロール                     | `j` キー または `↓` (下矢印キー) |                                        |
| 1行上へスクロール                     | `k` キー または `↑` (上矢印キー) |                                        |
| 表示されている内容の中で文字列を検索する | `/` の後に検索したい文字を入力して `Enter` | 下方向へ検索。見つかった場合、`n`で次候補、`N`で前候補へ移動 |
| 表示されている内容の中で文字列を逆方向に検索する | `?` の後に検索したい文字を入力して `Enter` | 上方向へ検索。操作は`/`と同様             |
| `less` を終了する                     | `q` キー                 |                                        |

![lessの基本操作](images/jless.png)
図5.3: `less` の主なキー操作

例えば、`スペース`キーを押すと次のページが表示され、`b`キーで前のページに戻ることができる。読み終わったら `q`キーで `less` を終了し、元のプロンプト表示に戻る。

画面の一番下にファイル名やコロン `:`、あるいはファイルの終端まで行くと `(END)` と表示されていれば、`less` が動作中である。`q` を押すまで他のコマンドは入力できない。

> ### ☆練習1: `less` の操作に慣れよう☆
>
> 1.  先ほど作成した `long_text.txt` を `less` で開いてみよう (`less long_text.txt`)。
> 2.  `スペース` キーで数ページ進んでみよう。
> 3.  `b` キーで数ページ戻ってみよう。
> 4.  `j` キーや `k` キーで1行ずつスクロールしてみよう。
> 5.  `/` に続けて `Linux` や `Darwin`（自分の環境に合わせて、表示されていそうな単語）を入力して `Enter` を押し、検索機能を試してみよう。
> 6.  検索で見つかった場合、`n` キーを押して次の候補へ移動できるか試してみよう。`N` キーで前の候補へ移動できるかも試してみよう。
> 7.  `q` キーで `less` を終了しよう。

### コマンドの実行結果も1ページずつ: `less` とパイプ

`less` は、ファイルの中身だけでなく、次のように「パイプ (`|`)」という機能と組み合わせることで、コマンドの実行結果も1画面ずつ区切って表示することができる。

```bash
「コマンド」 | less
```

例えば、前の手順で作成した `long_text.txt` ファイルの内容を `cat` コマンドで表示し、その結果を `less` で1画面ずつ表示してみよう。

```bash
% cat long_text.txt | less
```

UNIX系システムでは、「`|`」(パイプラインと呼ぶこともある) を使って、あるコマンドの実行結果 (標準出力) を、別のコマンドの入力 (標準入力) として渡すことができる。この仕組みを **「パイプ」** と言う。水道管が水を運ぶように、データが一方向へ流れていくイメージである。ここでは、`cat long_text.txt` の実行結果（つまり `long_text.txt` ファイルの内容）を、パイプを使って `less` コマンドの入力として渡し、1画面ずつ表示させている。これは、`less long_text.txt` とほぼ同じ結果になるが、パイプの動作を理解するための一例である。

パイプは非常に強力な機能で、複数のコマンドを連携させて複雑な処理を行う際によく使われる。

![lessとパイプ](images/jlesspipe.png)
図5.5: パイプ (`|`) を使って `cat` の出力を `less` に渡すイメージ

> ### ☆練習2: パイプと `less` を使ってみよう☆
>
> 1.  `ls -l /etc` というコマンドを実行すると、`/etc` ディレクトリ内のファイルやディレクトリの一覧が詳細情報と共にたくさん表示される。この結果を `less` で表示してみよう。
>     ```bash
>     % ls -l /etc | less
>     ```
> 2.  表示された内容を `スペース` キーや `b` キーでスクロールしてみよう。
> 3.  `/` を使って、例えば `conf` という単語を検索してみよう。
> 4.  `q` キーで `less` を終了しよう。

## ターミナルのスクロールバック機能も便利

実は、`less` を使わなくても、最近のターミナルソフト（macOSの「ターミナル.app」やWindowsの「ターミナル」など）には、画面をスクロールして過去の表示を遡って見る機能（スクロールバック機能）が標準で備わっている。

*   **macOS ターミナル**: マウスホイールやトラックパッドでスクロールできる。また、`Command + ↑` や `Page Up` キーなどでもスクロール可能である。
*   **Windows ターミナル**: こちらもマウスホイールやスクロールバーで操作できる。`Ctrl + Shift + ↑/↓` や `Page Up/Down` なども使える (設定による)。

短い出力結果であれば、このスクロールバック機能で十分なことも多い。しかし、非常に長い出力や、ファイルの内容をじっくり見たい場合、検索機能も使える `less` の方が便利な場面もある。状況に応じて使い分けよう。
