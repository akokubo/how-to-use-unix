# ディレクトリとパス

## ファイル・システムの全体像 {#ファイルシステムの全体像 .unnumbered}

UNIX
システムでは、ファイル・システムは、ツリー状の階層化ディレクトリ構造を
している。 これは簡単に言うと、

1.  たくさんあるファイルを、種類ごとに**「ディレクトリ」**と
    呼ばれる箱に分けてしまうことで整理する。

2.  ディレクトリの中にも、またディレクトリを作れる。

3.  ディレクトリの構造図を書くと、木が枝をひろげたよう(ツリー状)に見える。

ということだ。

このようなディレクトリ構造は、後に UNIX システムを真似した MS-DOS や
Windows などにも採り入れられた。 Windows
では、ディレクトリのことをフォルダと呼んでいる。

ところで、 UNIX システムには、いろいろなファイルやコマンドがあるが、
ディレクトリの中を自由に移動できるようになると、
これらのことをより詳しく知ることができるようになる。
また、ユーザ個人でも、自分のファイルをディレクトリに分類して格納しておくと、
ファイルが整理されて、とても便利だ。

この章では、ディレクトリの移動、作成、消去などについて紹介する。

## UNIX システムのディレクトリ構造

### ディレクトリの全体像

最初に UNIX システムの全体像を紹介してしまおう。 次のページの図

- **ルート・ディレクトリ /**

  一番上に、 `/` と書かれるルート・ディレクトリと呼ばれるものがある。
  「ルート -- root 」というのは「(木の)根」のことだ。 そして、その下に
  `home`、 `bin` 、 `etc` など、
  いろいろなディレクトリが、枝をひろげている。
  そして、それらのディレクトリの先にも、またディレクトリがあって、
  枝が更にひろがっている。 これがツリー状と言われる理由だ。
  絵としては、木が上下さかさまに描かれていると思ってくれればいい。

  では、ルート・ディレクトリの下を一つずつ説明していこう。

- **ホーム・ディレクトリ home**

  まず、 `/` の下には、 `home` というディレクトリがあり、
  この中に人の名前のついたディレクトリがある。 これらは、 UNIX
  システムで **「ホーム・ディレクトリ」**と呼ばれている。

  つまり、 `kokubo` さんのホーム・ディレクトリは、 `/` の下の `home`
  の下の `kokubo` だ。 なおこれは、 `/home/kokubo` と表す。
  要するに、ディレクトリの区切りマークが「`/`」というわけだ。

  また同様に、 `tomoda` さんのホーム・ディレクトリは、 `/` の下の `home`
  の下の `tomoda` で、 この場所のことを `/home/tomoda` と表す。

  login
  した直後、それぞれのユーザは、自分のホーム・ディレクトリに、まず入る。
  ここまでの演習で作ったファイルも、みんなここにある。

  例えば、 `kokubo` さんは、 login すると、 `/home/kokubo` に入る。
  そして、ここまでの演習で `kokubo` さんの作ったファイルは、
  ここに置いてあるということだ。

- **bin**

  `bin` というのは「バイナリ -- binary」の略で、本来は、
  文字ではないデータが入っているファイルのことだ。
  ちなみに、マシン語で書かれたコマンドや、画像、音声データなどが、
  本来の意味ではバイナリである。 UNIX
  システムでは、これらの内、コマンドを入れるためのディレクトリに `bin`
  を使う。

  `/` の下の `bin`、つまり `/bin` には、 UNIX
  システムの一番基本的なコマンドが置いてある。

- **etc**

  `etc` は「エトセトラ -- etc.」である。 UNIX
  システムでは、ここにシステムの各種設定用のファイルが置かれている。

- **tmp**

  `tmp` は「テンポラリ -- temporary」である。
  主に、システムが一時的にファイルを作るときに、ここを使う。

- **usr**

  `usr` は「ユーザ -- user」という意味だ。 この中は更に分かれていて、
  `/usr/bin` にはユーザが使う各種コマンド、 `/usr/include` には C
  言語のヘッダー・ファイルというものが、 `/usr/lib`
  にはユーザ・コマンドが使うライブラリというデータが 置かれている。

  また、 `/usr/local` 以下には、オプションのコマンドなどが
  置かれている。 これらは、 UNIX
  の基本システムには付いて来ないものだが、 便利なものも多い。 例えば、
  Mule などもここに置かれている。

  なお、この図には書いていないが、 `/usr/local` の下にも、 更に `bin` や
  `lib` などがある。

- **var**

  `var` は「変化する -- variable」という意味。 login
  の記録や、メールなどのシステムの記録などの、日々変化するものが置かれている。

## パス

UNIX システムでは、ツリー状のディレクトリ構造の中で、
ファイルやディレクトリの場所を指定するのに、 **パス** というものを使う。
「パス -- path」というのは、通り道や、経路という意味だ。
パスには、絶対パスと相対パスがある。

このパスをマスターすることで、好きな場所に移動して UNIX
の中を探検したり、
ファイルを好きな場所にコピーしたりすることができる[^1]ようになる。

パスは UNIX システムをマスターする上で、最重要ポイントの一つだ。

### 根っこからの通り道: 絶対パス

既に、ここまでのところで紹介してしまったが、 `/home/kokubo`のように、
[一番上のルート・]{.underline}\
[ディレクトリから、順番にディレクトリを書いて表す]{.underline}
方法を、**「絶対パス」**と呼ぶ。

図 [1.3](#absol){reference-type="ref" reference="absol"}
を使って説明しよう。 まず、一番上にルート・ディレクトリがあり、
これは絶対パスで表すと `/` になる。

そして、 `/` の下に `home` があって、 これは絶対パスでは `/home`
となる。

その下は学年ごとにディレクトリが用意されていて、 `ei99` は 絶対パスでは
`/home/ei99` となる。

その中に `ei99000` さんのホーム・ディレクトリがあって、 これは
`/home/ei99/ei99000` だ。

たとえば、 `ei99000` さんは、自分のホーム・ディレクトリの中に `Report`
というディレクトリを作ることができて、 これは
`/home/ei99/ei99000/Report` になる。

そして、またその中にディレクトリやファイルを作ることもできる。 たとえば
`11gatsu` というディレクトリを作ると、 これは絶対パスでは
`/home/ei99/ei99000/Report/11gatsu` になる。

このように根っこ(ルート・ディレクトリ)から順番に、書いていくのが絶対パスである。

なお、この図の例では、ディレクトリだけを書いたが、ファイルの場合も全く同様だ。
`ei99000` さんのホーム・ディレクトリ `/home/ei99/ei99000` に `ThisMonth`
というファイルがある場合、 このファイルは絶対パスでは
`/home/ei99/ei99000/ThisMonth` と書く。

### 自分がいるところが中心: 相対パス

絶対パスを使って、ルート・ディレクトリから、順番書いていけば、
どんなディレクトリでも表せる。
しかし、絶対パスだと、だんだんディレクトリが下の方になってくると、
どんどん長くなって面倒だ。
しかも、自分のホーム・ディレクトリの中にあるファイルまで、わざわざ長々と指定
しなければならない。

そこで、 UNIX システムでは、**「相対パス」**という、
もう一つの方法が用意されている。
相対パスでは、俺が世界の中心という感じで、自分が今いるところを基準にして
指定する。

では、図 [1.4](#relative){reference-type="ref" reference="relative"}
を見ながら説明しよう。
これは、さっき絶対パスの説明に使ったものと全く同じだ。

まず、 `ei99000` さんは、今自分のホーム・ディレクトリにいるとしよう。

この下に、 `Report` というディレクトリがある。
これは、相対パスでは、単に `Report` である。 絶対パスだと、わざわざ
`/home/ei99/ei99000/Report` と長くて面倒だが、
相対パスではこんなに短くなる。

それから、 `Report` の下の `11gatsu` も、 `Report/11gatsu`
ととても簡単になる。

このように相対パスとは、
[自分が今いるところから、順番にパスを書いていく]{.underline}方法である。

<figure id="relative">

<figcaption>相対パスので表し方</figcaption>
</figure>

この例でわかるように、自分がいるところより下の方は、
「相対パス」を使うと、「絶対パス」よりも簡単に表わせる。

では、逆に上の方はどうか？

UNIX システムでは、 1 つ上のディレクトリを「`..`」と表す。
つまり、自分のホームの一つ上 `/home/ei99` は、単に `..` となる。

この**「ピリオド 2 つで、 1 つ上」**は、とてもよく使うので覚えておこう。

では、 2 つ上はどうか？

1 つ上が「`..`」で、ディレクトリの区切りが「`/`」なので、 2
つ上は「`../..`」となる。 つまり、 `/home` は、相対パスでは `../..`
となる。

同様にルート・ディレクトリ `/` は、相対パスでは `../../..` である。

最後にもう一つ。 **自分が今いる場所は、相対パスでは「`.`」**と、
ピリオド 1 つで表わす。 これもよく使うので覚えておこう。

つまり、今 `/home/ei99/ei99000` にいるなら、ここは相対パスで 表すと `.`
となる。

なお、以上の例は、あくまで `/home/ei99/ei99000` にいる場合の話だ。
相対パスは自分が今いるところを中心に指定するので、
別なディレクトリにいる場合、話が全く変わってくる。

例えば、 `/home/ei99/ei99000/Report` にいる場合、
自分のホーム・ディレクトリは一つ上にあるので、相対パスでは `..` となる。
`11gatsu` は、今いるところのすぐ下にあるので、相対パスでは `11gatsu`
になる。
このように相対パスは、現在どこのディレクトリに自分がいるかによって変わってしまう。

<figure id="relative2">

<figcaption><code>Report</code> の中にいるときの相対パス</figcaption>
</figure>

UNIX システムでは、絶対パスで指定しても、相対パスで指定しても、
効果は全く同じで、自分で好きな方を使っていい。
ただ、自分が今いるところの近くを指定するには「相対パス」を、
ルート・ディレクトリの近くを指定するには「絶対パス」を使った方が簡単だ。
このように使い分けるので、両方を知っておいた方が便利だ。

### ファイルの移動、コピー

`mv` や `cp` で別のディレクトリにファイルを移動したり、
コピーする方法を紹介しよう。

例えば、 `ThisMonth` というファイルを `CAL` という
ディレクトリに移動するには次のようにする。

まず、 `CAL` というディレクトリがあるかどうか確認する。

::: progls
1998nen 2000nen ThisMonth Years 1999nen RMAIL ThisYear exam1
:::

すると、上のようにないことがわかる。 そこで `CAL` を `mkdir` で作る。

::: progls
:::

ファイルを移動やコピーする場合でも、 `mv` や `cp` の使い方は、
普通に移動したりコピーする場合と全く同じだ。
ただ、ファイル名として、パスを書いてやればいい。

つまり、 `ThisMonth` を `CAL` に移動するには、
`mv ThisMonth CAL/ThisMonth` である。

なお、これは、 `mv ThisMonth CAL/.` とも書ける。
「`.`」をファイル名の代わりに使うと、「同じ名前で」という意味になるのだ。

このように「`.`」は、ディレクトリのときには「現在の場所」、
ファイル名のときには「同じ名前で」という意味になるので、混同しないように
注意しよう。

では、実際にやってみる。

::: progls
:::

`ls -F` で確かめてみると、たしかに `ThisMonth` がないことがわかる。

::: progls
1998nen 2000nen RMAIL Years 1999nen CAL/ ThisYear exam1
:::

`CAL` に入って中を見て見ると、 `ThisMonth` があることがわかる。

::: progls
ThisMonth
:::

なお、これは `mv` を使った例だが、代わりに `cp` を使えば、
コピーになる。  

[^1]: セキュリティ上、 いじれないようになっている場所もある。
